# BPCells Compatibility Test Suite

Plan for verifying a native Rust BPCells reader against the reference
R implementation.  The test suite comes before the reader — fixtures are
generated first so that the decoder is written test-first.

---

## Goals

1. Confirm byte-exact understanding of the on-disk format before writing a
   single line of decoder.
2. Cover every codec variant (unpacked, BP-128-FOR, BP-128-D1Z) with a
   minimal hand-checkable case.
3. Provide a regression harness so format changes in a future BPCells release
   are caught immediately.

---

## References

- Format spec: https://bnprks.github.io/BPCells/articles/web-only/bitpacking-format.html
- Reference bitpacking implementation (scalar C++, no SIMD):
  https://github.com/GreenleafLab/BPCells_paper/blob/main/utils/bitpacking-reference-implementation.cpp
- `StoredMatrix.h` — which files are opened and how:
  `r/src/bpcells-cpp/matrixIterators/StoredMatrix.h`
- `binaryfile.h` — 8-byte ASCII header format:
  `r/src/bpcells-cpp/arrayIO/binaryfile.h`

## Fixture inventory

All fixtures live under `tests/golden/bpcells/`.  Each is a directory
written by the R generator script below.

### Synthetic (always present, committed to the repo)

These are tiny (≤ 10 cells × 10 genes) so committing the raw binary is fine.

| Directory | Description |
|-----------|-------------|
| `synth_packed_uint_csc/`   | 4×5 uint32 CSC, packed — the canonical Seurat v5 path |
| `synth_unpacked_uint_csc/` | same matrix, unpacked — baseline / sanity check |
| `synth_packed_float_csc/`  | 4×5 float32 CSC, packed — floats keep `val` uncompressed |
| `synth_packed_double_csc/` | 4×5 float64 CSC, packed — double variant |
| `synth_packed_uint_csr/`   | 4×5 uint32 CSR (storage_order = "row") |
| `synth_empty/`             | 0-nnz matrix — idxptr all zeros, no val/index data |
| `synth_one_col/`           | 1-column matrix — edge case for nextCol() boundary |
| `synth_large_vals/`        | values up to 2^31−1 — exercises full 31-bit encoding |
| `synth_128_boundary/`      | exactly 128 nnz in one column — chunk boundary alignment |
| `synth_256_boundary/`      | exactly 256 nnz — two full chunks |

### Known matrix (generated from pbmc3k, optional)

| Directory | Description |
|-----------|-------------|
| `pbmc3k_packed_uint_csc/` | PBMC 3k, uint32, packed CSC — real-world scale |

Generated by `tests/generate_bpcells_fixtures.R` when the golden H5Seurat
fixture is present.  Skipped gracefully in CI if not.

## Fixture generator (`tests/generate_bpcells_fixtures.R`)

```r
library(BPCells)
library(Matrix)

out <- "tests/golden/bpcells"
dir.create(out, showWarnings = FALSE)

# ── helpers ────────────────────────────────────────────────────────────────
write_both <- function(mat, name_packed, name_unpacked) {
  bm <- as(mat, "IterableMatrix")
  write_matrix_dir(bm, file.path(out, name_packed),   compress = TRUE)
  write_matrix_dir(bm, file.path(out, name_unpacked), compress = FALSE)
}

# ── synthetic 4×5 matrix ───────────────────────────────────────────────────
#   col:   1   2   3   4   5
# row 1:   1   0   3   0   0
# row 2:   0   2   0   4   0
# row 3:   5   0   0   0   6
# row 4:   0   0   7   0   8
raw <- Matrix::sparseMatrix(
  i = c(1,3,2,1,2,4,2,3,4),
  j = c(1,1,2,3,4,3,4,5,5),
  x = c(1,5,2,3,4,7,4,6,8),
  dims = c(4, 5)
)

write_both(raw, "synth_packed_uint_csc", "synth_unpacked_uint_csc")

# float variant
write_matrix_dir(
  as(raw * 1.5, "IterableMatrix"),
  file.path(out, "synth_packed_float_csc"), compress = TRUE
)

# double variant — BPCells writes double when input is numeric (not integer)
write_matrix_dir(
  convert_matrix_type(as(raw, "IterableMatrix"), "double"),
  file.path(out, "synth_packed_double_csc"), compress = TRUE
)

# CSR variant
write_matrix_dir(
  t(as(t(raw), "IterableMatrix")),   # force row-major storage
  file.path(out, "synth_packed_uint_csr"), compress = TRUE
)

# empty matrix
write_matrix_dir(
  as(Matrix::sparseMatrix(i=integer(0), j=integer(0), x=integer(0),
                          dims=c(4,5)), "IterableMatrix"),
  file.path(out, "synth_empty"), compress = TRUE
)

# single column
write_matrix_dir(
  as(raw[, 1, drop=FALSE], "IterableMatrix"),
  file.path(out, "synth_one_col"), compress = TRUE
)

# large values (close to 2^31-1 — R integer max)
large <- Matrix::sparseMatrix(
  i = c(1,2,3,4), j = c(1,2,3,4),
  x = c(2L^31L - 1L, 2L^30L, 1L, 1000000L),
  dims = c(4, 4)
)
write_matrix_dir(
  as(large, "IterableMatrix"),
  file.path(out, "synth_large_vals"), compress = TRUE
)

# exactly 128 nnz in column 1 — chunk boundary
m128 <- Matrix::sparseMatrix(
  i = 1:128, j = rep(1L, 128), x = 1:128, dims = c(128, 2)
)
write_matrix_dir(
  as(m128, "IterableMatrix"),
  file.path(out, "synth_128_boundary"), compress = TRUE
)

# exactly 256 nnz — two full chunks
m256 <- Matrix::sparseMatrix(
  i = 1:256, j = rep(1L, 256), x = 1:256, dims = c(256, 2)
)
write_matrix_dir(
  as(m256, "IterableMatrix"),
  file.path(out, "synth_256_boundary"), compress = TRUE
)

# ── pbmc3k (optional) ──────────────────────────────────────────────────────
pbmc_h5 <- "tests/golden/pbmc3k.h5seurat"
if (file.exists(pbmc_h5)) {
  pbmc <- SeuratDisk::LoadH5Seurat(pbmc_h5)
  bm <- as(pbmc[["RNA"]]$counts, "IterableMatrix")
  write_matrix_dir(bm, file.path(out, "pbmc3k_packed_uint_csc"), compress = TRUE)
  message("pbmc3k fixture written")
} else {
  message("SKIP: pbmc3k.h5seurat not found")
}

message("Done. Fixtures written to ", out)
```

## Rust test structure

### Unit tests: codec algorithms (`crates/scx-core/src/bpcells.rs`)

These tests do not touch the filesystem — they verify the bitpacking
algorithms in isolation using hand-computed expected values.

```
test_bp128_decode_zeros         — all-zero block (0 bits per value)
test_bp128_decode_ones          — all-one block (1 bit per value)
test_bp128_decode_full_32bit    — values requiring 32 bits
test_bp128_decode_128_boundary  — exactly one full chunk
test_bp128_decode_256_boundary  — two full chunks, check carry-over
test_bp128_for_decode           — BP-128-FOR: val-1 round-trip
test_bp128_d1z_decode_sorted    — sorted indices: zigzag rarely fires
test_bp128_d1z_decode_unsorted  — non-monotone run: zigzag fires
```

Each test:
1. Constructs the packed representation by hand (or borrows from the
   reference implementation).
2. Calls the Rust decoder.
3. Asserts exact equality with the expected u32 values.

### Integration tests: directory reader (`crates/scx-core/tests/bpcells_compat.rs`)

One test per fixture.  Each test:
1. Calls `BpcellsReader::open(path)`.
2. Iterates column by column via `nextCol()` / `load()`.
3. Reconstructs a dense matrix.
4. Asserts against a hardcoded expected dense matrix (for synthetic fixtures)
   or against a checksum / nnz count (for pbmc3k).

```
test_synth_packed_uint_csc      — full value/index check
test_synth_unpacked_uint_csc    — same matrix, unpacked path
test_synth_packed_float_csc     — float uncompressed val path
test_synth_packed_double_csc    — double variant
test_synth_packed_uint_csr      — CSR storage_order handling
test_synth_empty                — 0-nnz: no panics, shape correct
test_synth_one_col              — boundary: single column
test_synth_large_vals           — 2^31-1 round-trip
test_synth_128_boundary         — exactly one full BP-128 chunk
test_synth_256_boundary         — two full BP-128 chunks
test_pbmc3k_packed_uint_csc     — skipped if fixture absent; checks nnz count
```

### Cross-check test

One additional test reads `synth_packed_uint_csc` with `BpcellsReader` and
also reads `synth_unpacked_uint_csc` with the same reader.  It asserts that
the two resulting matrices are identical — packed and unpacked paths must
agree.

## Version probe: what files to expect

Before writing any decoder, the test for `synth_packed_uint_csc` should start
by just inspecting the directory tree and binary headers produced by R:

```rust
// Proto-test: print every file, its size, and its 8-byte header
// Run with: cargo test -p scx-core inspect_bpcells -- --nocapture
#[test]
fn inspect_bpcells_packed_uint() {
    let dir = Path::new("../../tests/golden/bpcells/synth_packed_uint_csc");
    if !dir.exists() { return; }
    for entry in std::fs::read_dir(dir).unwrap() {
        let e = entry.unwrap();
        let path = e.path();
        let meta = path.metadata().unwrap();
        let mut hdr = [0u8; 8];
        if let Ok(mut f) = std::fs::File::open(&path) {
            let _ = std::io::Read::read(&mut f, &mut hdr);
        }
        println!("{:20} {:8} bytes  header={:?}", e.file_name().to_str().unwrap(), meta.len(), std::str::from_utf8(&hdr));
    }
}
```

Running this (after generating fixtures) tells us exactly what BPCells 0.3.x
writes, which we can compare against the spec.

## Acceptance criteria

The test suite is complete when:

- All 10 synthetic fixtures exist and pass.
- The `inspect_bpcells_packed_uint` probe test prints a file tree that matches
  the spec (correct file names, expected header bytes).
- The packed/unpacked cross-check passes.
- The pbmc3k test (when the fixture exists) reports the correct nnz count
  (2,286,884 for the standard PBMC 3k dataset).

At that point the decoder is trustworthy and `H5SeuratReader` v5 routing can
be wired up.

## Open questions

1. **`storage_order` in `.h5seurat`**: When Seurat v5 embeds a BPCells matrix
   inside an HDF5 file (rather than a directory), the version string is an
   HDF5 attribute and arrays are HDF5 datasets.  Does `H5SeuratReader` see
   packed arrays inside the HDF5 group, or only directory-backed ones?
   → Need a real Seurat v5 `.h5seurat` file to check.

2. **v1 vs v2 `idxptr`**: v1 used uint32 for `idxptr`; v2 uses uint64.  The
   `version` file distinguishes them.  Synthetic fixtures will use v2 (current
   BPCells), but the reader should handle v1 for old files.

3. **Float `val` packing**: The spec says float/double `val` is always stored
   uncompressed even in the `packed` variant.  `StoredMatrix.h` confirms this
   via the `if constexpr (std::is_same_v<T, uint32_t>)` branch.  The Rust
   reader should assert this assumption rather than silently misreading.
